## 1. 背景：我们要解决什么问题？

### 1.1 手写数字识别是什么？

想象你面前有一堆手写的邮政编码信封。每张信封上有一个手写的数字（0 到 9），你的任务是**让电脑自动认出是哪个数字**。

```
┌─────────────┐    ┌──────────────────┐    ┌──────┐
│ 手写数字图片  │ →  │ 深度学习模型(CNN)  │ →  │ "5"  │
│  (28×28 灰度) │    │   (LeNet-5)      │    │ (预测结果)│
└─────────────┘    └──────────────────┘    └──────┘
```

这个问题在 1990 年代具有巨大的商业价值——美国邮政系统每天要处理数百万封信件，自动识别邮政编码可以节省大量人力成本。

### 1.2 MNIST 数据集是什么？

MNIST 是机器学习领域最著名的"Hello World"级数据集。你可以把它想象成一个**标准题库**：

| 属性 | 数值 | 类比 |
|---|---|---|
| 样本总数 | 70,000 张 | 题库共有 7 万道题 |
| 训练集 | 60,000 张 | 6 万道用来学习和练习 |
| 测试集 | 10,000 张 | 1 万道用来期末考试 |
| 图片尺寸 | 28×28 像素 | 每张图只有 784 个小格子 |
| 颜色 | 灰度（黑和白之间的不同深浅） | 没有彩色，只有"有多黑" |
| 类别 | 0, 1, 2, ..., 9（共 10 类） | 答案只有 0 到 9 十种可能 |

**一个关键认知**：对电脑来说，一张 28×28 的图片就是一个 **784 个数字的列表**。每个数字代表对应像素的亮度（0 = 全黑，255 = 全白）。

```
人眼看到的 "5"：          电脑看到的（28×28 数字矩阵）：
  ███                     0  0  0 150 255 200  50  0  ...
  █                       0  0  200 255 180  0  0  0  ...
  ████                    0  0  0  150 255 210  50  0  ...
     █                    ...（共 784 个数字，每个 0–255）
  ████
```

## 2. 核心概念：用生活类比理解神经网络

### 2.1 什么是神经元？

一个神经元做的事情很简单：**接收一些输入 → 加权求和 → 决定是否"激活"**。

生活类比：你决定今天要不要出门。

- 输入因素：天气好不好？（0–10 分）、有没有约？（0/1）、累不累？（0–10 分）
- 每个因素有不同**权重**：天气权重 0.5、有约权重 1.0、疲劳权重 −0.8
- 如果加权总分超过某个**阈值**，你就出门；否则不出门

```
输入₁ × 权重₁ + 输入₂ × 权重₂ + 输入₃ × 权重₃ → 是否超过阈值 → 输出 0 或 1
```

神经网络的"学习"过程，就是**不断调整这些权重**，让输出越来越接近正确答案。

### 2.2 什么是层？

单个神经元能力有限（只能做简单的是/否判断）。把很多神经元叠起来，形成"层"：

```
输入层(784个数字) → 隐藏层₁ → 隐藏层₂ → ... → 输出层(10个数字，代表0–9的概率)
```

类比：工厂流水线
- **第 1 道工序**：检查有没有横线
- **第 2 道工序**：检查有没有圆圈
- **第 3 道工序**：根据横线和圆圈的组合判断是几

每道工序就是一层神经元。

### 2.3 什么是卷积？—— 最关键的概念

普通神经网络（全连接层）有一个致命问题：它把 28×28 的图片**直接拉直成一个 784 个数字的长列表**。

```
普通网络看到：  [数字0, 数字1, 数字2, ..., 数字783]
                ↑ 完全不知道哪些像素是相邻的！
```

**卷积层**不同。它用一个小的"滑块窗口"（比如 5×5）在图片上一格一格地滑动，每次只看窗口内的一小片区域。

```
生活类比：你用放大镜检查一张照片
┌─────────────────────────┐
│    ┌───┐                │
│    │窗口│ ← 每次只看5×5   │  这个"窗口"就是卷积核
│    └───┘                │  它会从左上角滑到右下角
│                         │
│               →→→→→     │  每滑一步，算一个值
│               →→→→→     │
│               →→→→→     │
└─────────────────────────┘
```

**卷积核**的内容是学出来的：比如一个核学会检测"→"方向的边缘，另一个核学会检测"○"形状。

**为什么这很重要？** 因为手写数字的特征是局部的：一个"8"有两个圈，"7"有一条横线和一条斜线。卷积天然适合捕捉这些局部模式。

### 2.4 什么是池化？—— 压缩信息

卷积之后，我们得到了很多"特征图"。但这些图太大了——池化就是**每 2×2 的区域只保留最重要的那一个值**，把图缩小一半。

| 池化方式 | 怎么做 | 效果 |
|---|---|---|
| **最大池化 (MaxPool)** | 取 2×2 中最大的值 | "这个区域有没有强特征？" |
| **平均池化 (AvgPool)** | 取 2×2 的平均值 | "这个区域整体特征水平如何？" |

```
原始 4×4 特征图     2×2 MaxPool 后      2×2 AvgPool 后
┌───────────┐      ┌─────┬─────┐      ┌─────┬─────┐
│ 1  3  2  1│      │     │     │      │     │     │
│ 2  6  4  2│  →   │  6  │  5  │      │ 3.0 │ 2.8 │
│ 3  1  5  1│      ├─────┼─────┤      ├─────┼─────┤
│ 4  2  2  3│      │  4  │  5  │      │ 2.5 │ 2.8 │
└───────────┘      └─────┴─────┘      └─────┴─────┘
```

### 2.5 训练是什么？—— Loss 和优化器

**Loss（损失）**：模型当前预测和正确答案之间的"差距"。差距越大，loss 越大。训练的目标就是让 loss 越来越小。

```
生活类比：学射箭
- 第 1 箭：偏了 30 厘米（loss = 30）
- 第 2 箭：偏了 15 厘米（loss = 15）← 有进步
- 第 3 箭：偏了 5 厘米（loss = 5）  ← 继续进步
- ...
- 第 N 箭：正中靶心（loss ≈ 0）
```

**优化器（Optimizer）** 决定每次"往哪个方向、多大步地"调整权重。

| 优化器 | 特点 | 类比 |
|---|---|---|
| **Adam** | 自动调节步长，收敛快 | 有 GPS 导航的自动驾驶 |
| **SGD + Momentum** | 固定步长 + 惯性 | 手动驾驶但有定速巡航 |

## 3. LeNet-5：1998 年的经典模型

### 3.1 结构总览（一张图看懂）

```
输入: 28×28 灰度图（784 个数字）
    │
    ▼
┌─────────────────────────────────────────────────┐
│  C1: 卷积层 (6个 5×5 卷积核)    输出: 6@28×28     │  ← 检测边缘、角点
│  S2: 池化层 (2×2 取最大)         输出: 6@14×14     │  ← 压缩一半
├─────────────────────────────────────────────────┤
│  C3: 卷积层 (16个 5×5 卷积核)   输出: 16@10×10    │  ← 检测局部形状
│  S4: 池化层 (2×2 取最大)         输出: 16@5×5      │  ← 再压缩一半
├─────────────────────────────────────────────────┤
│  C5: 卷积层 (120个 5×5 卷积核)  输出: 120@1×1     │  ← 全局特征
├─────────────────────────────────────────────────┤
│  F6: 全连接层 (120 → 84)         输出: 84         │  ← 特征整合
│  Output: 全连接层 (84 → 10)      输出: 10         │  ← 最终分类
└─────────────────────────────────────────────────┘
    │
    ▼
输出: [0.01, 0.00, 0.02, ..., 0.89, ...]
       数字0  数字1  数字2       数字8
       ↑ 概率 0.01              ↑ 概率 0.89 → 预测为 "8"
```

**总参数量：61,706 个**（现代模型动辄数亿参数，LeNet-5 是真正的"微型"网络）

### 3.2 本项目和原论文的区别

| 组件 | 1998 年原论文 | 本项目 | 为什么改？ |
|---|---|---|---|
| 激活函数 | **Tanh**（S 型曲线） | **ReLU**（负值直接归零） | Tanh 容易"学不动"，ReLU 更快更好 |
| 池化 | **AvgPool**（取平均） | **MaxPool**（取最大） | MaxPool 是现代标配，但我们也做了对比实验 |
| 损失函数 | **MSE**（均方误差） | **CrossEntropyLoss** | 后者更适合分类任务 |
| 优化器 | **SGD**（手动调学习率） | **Adam**（自适应） | Adam 收敛更快，但我们也对比了 SGD |
| C3 连接表 | **稀疏连接**（不全部连） | **全连接** | 简化实现，现代惯例 |

## 5. 实验全景：我们做了 8 组对照实验

### 5.0 为什么做对照实验？

科学的本质是**控制变量**——每次只改变一个因素，看它如何影响结果。这就好比：

- "加糖之前 vs 加糖之后，咖啡味道差多少？" —— 除了糖以外，所有条件不变
- "用 Adam vs 用 SGD 训练，准确率差多少？" —— 除了优化器以外，所有条件不变

### 5.1 实验矩阵总览

| 编号 | 实验名称 | 我们在比什么？ | 结果（简化版） |
|---|---|---|---|
| Exp-1 | **池化方法** | MaxPool vs AvgPool | 差异仅 0.04%，可以忽略 |
| Exp-2 | **优化器** | Adam vs SGD | SGD 略优 0.07%，但差距很小 |
| Exp-3 | **激活函数** | ReLU vs Tanh | ReLU 好 0.28%，Tanh 更易过拟合 |
| Exp-4 | **多种子稳定性** | 3 个种子 × 3 轮 = 9 次 | 一致性极高，标准差仅 0.07% |
| Exp-5 | **学习率** | lr = 0.01 / 0.001 / 0.0001 | 0.001 是最优值 |
| Exp-6 | **CNN vs MLP** | 卷积 vs 全连接 | CNN 错误率仅为 MLP 的 37% |
| Exp-7 | **数据增强** | 有无数据增强 | 增强后 99.21%，错误减少 17.7% |
| Exp-8 | **Batch Size** | bs = 16 / 64 / 256 | 64 是最优平衡点 |

### 5.2 各组实验逐一解读

#### Exp-1：池化方法对比（MaxPool vs AvgPool）

**问题**：原论文用 AvgPool（平均值），现代用 MaxPool（最大值）。到底哪个好？

**结果**：MaxPool 99.04% vs AvgPool 99.00%，**差距仅 0.04%**。在 10,000 张测试图里只差了 4 张。

**结论**：**两者在 MNIST 上效果差不多，不用纠结。**

#### Exp-2：优化器对比（Adam vs SGD）

**问题**：Adam 是 2014 年才提出的自适应优化器，比原论文的 SGD 更现代。哪个好？

**结果**：SGD (lr=0.01) 99.11% vs Adam 99.04%。SGD 反而高了 0.07%。

**注意**：SGD 需要 lr=0.01，如果也用 Adam 的 lr=0.001，SGD 只能到 98.68%。**SGD 需要更大的学习率。**

**结论**：在这个简单任务上，两个优化器效果相当。SGD 没有过时。

#### Exp-3：激活函数对比（ReLU vs Tanh）

**问题**：原论文用 Tanh，现代用 ReLU。ReLU 到底好在哪？

**结果**：ReLU 99.04% vs Tanh 98.76%。差距 0.28%。

**深层分析**：Tanh 的问题不是 "一开始就慢"（epoch 1 两人打平），而是"**后期过拟合**"——训练集上继续进步，测试集上停滞。这说明 Tanh 更容易记住训练集的噪声。

**结论**：ReLU 确实更好，但不是"碾压级"的。在浅层网络上，Tanh 也能用。

#### Exp-4：多种子稳定性（9 次重复实验）

**问题**：我们的结果（99.04%）是否可靠？再跑一次还能得这个数吗？

**方法**：用 3 个不同随机种子，每个跑 3 遍，共 9 次。

| 种子 | 第 1 次 | 第 2 次 | 第 3 次 | 均值 |
|---|---|---|---|---|
| 42 | 99.04% | 98.95% | 98.91% | 98.97% |
| 123 | 99.08% | 99.07% | 99.07% | 99.07% |
| 456 | 99.03% | 99.10% | 99.15% | 99.09% |

**结论**：9 次结果集中在 98.91%–99.15%，标准差仅 0.07%。**结果高度可信。**

#### Exp-5：学习率对比

**问题**：学习率 0.001 是否最优？大一点或小一点会怎样？

**结果**：

| lr | 准确率 | 现象 |
|---|---|---|
| 0.01（太大） | 98.24% | 步子太大，在最优解附近反复横跳 |
| 0.001（刚好） | **99.04%** | 平稳收敛 |
| 0.0001（太小） | 98.44% | 走得慢，10 轮来不及学完 |

**理解**：lr 太高就像"下山时步子太大，跨过了山谷"；lr 太低就像"蚂蚁爬下山，太慢"。

#### Exp-6：CNN vs MLP（最震撼的对比）

**问题**：不用卷积，直接用全连接层，效果差多少？

**方法**：MLP 有约 59,000 个参数（和 LeNet-5 的 61,706 基本持平）。

**结果**：CNN 99.04% vs MLP **97.38%**。

| 指标 | CNN (LeNet-5) | MLP |
|---|---|---|
| 错误率 | 0.96% | 2.62% |
| 错误样本数 | 96 / 10,000 | 262 / 10,000 |

**CNN 的错误只有 MLP 的 37%**。也就是说，用同样的参数量，CNN 少犯了 166 个错误。

**为什么？** MLP 把图片拉成一条直线，完全不知道"哪些像素相邻"。CNN 的卷积核天然理解空间结构。

**这是本报告最有力的结论**：证明了卷积结构的必要性。

#### Exp-7：数据增强

**问题**：对训练图片做一些微小变换（旋转 ±10°、平移 10%），能让模型更强吗？

**结果**：99.04% → **99.21%**（错误率从 0.96% 降到 0.79%，减少 17.7%）。

**意义**：这是本项目**最高准确率**。也验证了原论文 1998 年就使用的数据增强策略在今天仍然有效。

#### Exp-8：Batch Size 对比

**问题**：一次给模型看多少张图同时训练，会影响效果吗？

**结果**：bs=64 是最优的。bs=16 梯度噪声太大，bs=256 更新频率太低。但在 ±0.11% 范围内，影响不大。

## 附录 B：常见困惑解答

**Q: 为什么模型会认错数字？**

A: 回想一下你自己写字——有时候你的 "9" 写得像 "4"，"2" 写得像 "7"。模型也在面对同样的问题：手写体的类内差异和类间相似性。

**Q: 99% 是不是意味着模型比人还厉害？**

A: 不一定。MNIST 的 1% 错误率中，很多是人类也无法区分的（潦草、残缺、歧义）。而且如果给人类同样的 28×28 低分辨率图，人类可能也有 0.2%–0.5% 的错误率。

**Q: 为什么不做更炫酷的模型？**

A: 这是**论文复现项目**，目标不是追求最高分，而是理解经典和训练工程能力。LeNet-5 是 CNN 的"地基"，理解了它才能理解 ResNet、Transformer 等现代架构。

**Q: PyTorch 和 TensorFlow 哪个好？**

A: 本项目用 PyTorch，因为代码风格更像 Python，适合学习和实验。学术界目前 PyTorch 占主流（70%+）。
